Forms-50 does. 50 production forms across 5 categories — government, financial, healthcare, commerce, hiring. Public leaderboard, open methodology, one persona, five axes.

02

Methodology

RULE 01
Fill, don't submit
Agents fill every required field on the form using a fixed set of test data. Submit is only clicked on demo or sandbox forms. We grade fillability, not actual submission. No real accounts touched.
RULE 02
Fixed test persona
Every run uses the same persona: Test User, 1234 Example St, San Francisco CA 94110, DOB 1990-01-15, phone 415-555-0100, email test@example.com. Listed in full in the GitHub repo.
RULE 03
Five scoring axes
Completion — % of forms reached the final review step. Accuracy — % of required fields filled with the expected value. Time — wall-clock seconds from start to last action. Cost — token + API spend per run. Failure mode — what broke when it broke.
RULE 04
Failure mode tags
Stuck classifications: date-picker, multi-select, conditional-fields, captcha, iframe, file-upload, multi-step-nav, auth-wall, infinite-loop, wrong-value. Used to compare agents on what they specifically can't do.
RULE 05
One run per agent per form
Each agent gets a single attempt per form. Some forms have variance — captchas appear sometimes, edge cases trigger differently. Variance is captured in failure tags, not retries. v0.2 will introduce a 3-run average.
RULE 06
Open submission
Anyone can run their agent against the 10 forms and submit results. Submitted runs marked "self-reported" until verified. Verified runs are observed live and timestamped. Submission link at the bottom of this page.

The 10 forms

Built an agent? Submit a run.
