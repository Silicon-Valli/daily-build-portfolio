Repo Decoder
FREE PREVIEW

Paste any GitHub URL, get a plain-English breakdown. What it does, who it's for, key stats, and a copyable pitch. No backend, no API key, calls GitHub's public API.

Stack: single-file HTML ~4 hours Monetization: email list + $9/mo digest
The demand signal

Vibe coding put GitHub at the center of how builders discover tools. But GitHub is built for developers. Non-technical founders, PMs, and investors constantly hit repos they can't parse. "What does this actually do?" is a real question that wastes real time. Current workflow: read 3,000 lines of README or skip entirely.

Exact build steps
  1. Single HTML file. No build step, deploys to Vercel by drag-and-drop.
  2. Input parsing: regex github.com/([^/]+)/([^/\s?#]+), strip .git suffix.
  3. Two parallel API calls via Promise.allSettled: GET /repos/{owner}/{repo} and GET /repos/{owner}/{repo}/readme.
  4. README parsing: decode base64 with atob(), strip code blocks, headings, image/badge lines, HTML tags. Take first 2-3 clean paragraphs, cap at 500 chars.
  5. Audience inference: regex patterns against description + topics to classify target users.
  6. Render: four cards — header with stats, "What it does", "Who it's for", copyable pitch.
  7. Error states: 404, 403 (rate limit), network fail. All inline, never break the page.
  8. Share URL routing: ?repo=owner/name auto-loads on page open. Makes every decoded repo shareable.
Monetization path

Immediate (free): post in r/SideProject and r/vibecoding. People share repo links with their teams using the generated pitch text — built-in distribution.

Short-term: email capture after 3 uses. Grow a list of non-technical founders who regularly evaluate GitHub repos.

Medium-term: "Repo Radar" — weekly $9/mo digest of the 10 most-starred repos from the past 7 days, each with the plain-English breakdown auto-generated.

Longer-term: B2B angle. "Repo audit" for procurement/security teams evaluating open-source dependencies. $49/mo per team.
